## 任务清单

### 任务 E-1：代码通读与问题发现

- **目标**：深入理解分享码和备份系统的代码逻辑，找到改进点
- **预计时间**：30分钟
- **具体步骤**：
  1. 阅读 `js/app.js` 中 `_generateShareCode()`（L1093-1095）：理解分享码编码逻辑（JSON → Base64 → 截取）
  2. 阅读 `importShareCode()`（L1116-1152）：理解导入解码 + 配置填充流程
  3. 阅读 `_renderShares()`（L1154-1160）：理解分享列表渲染
  4. 阅读 `exportBackup()`（L455-470）和 `importBackup()`（L472-500）
  5. 在浏览器中实际操作生成分享码 → 在新标签页导入 → 验证一致性
  6. 记录至少2个可改进问题
- **验收标准**：写出对分享码/备份数据流的理解，列出至少2个改进点

---

### 任务 E-2：分享码错误处理增强

- **目标**：为 `importShareCode()` 增加分类错误提示
- **预计时间**：30分钟
- **具体步骤**：
  1. 当前 `importShareCode()` 的错误处理较粗粒度（仅"未找到该分享码"）
  2. 增加以下分类错误提示：
     - 输入为空：`'请输入分享码'`
     - 格式无效：`'分享码格式不正确，请检查'`
     - 解码失败：`'分享码已损坏，无法解析'`
     - 重复导入（与已有 agent 配置相同）：`'该配置已存在'`
  3. 错误信息通过 `$('#shareError').textContent = '...'` 展示
  4. 每种错误场景都通过 `Toast.show()` 给出视觉反馈
- **验收标准**：输入不同异常值时，显示对应的明确错误提示

---

### 任务 E-3：分享码去重逻辑

- **目标**：避免同一配置重复生成多条分享记录
- **预计时间**：20分钟
- **具体步骤**：
  1. 阅读 `shareCurrentConfig()`（L1098-1114）：当前每次保存配置都会生成新分享码
  2. 修改逻辑：在生成新分享码前，检查 `Storage.get('shares', [])` 中是否已存在内容相同的分享
  3. 如果已存在（对比 agent name + gender + presets），不重复添加，仅提示"分享码已存在：XXX"
  4. 如果配置有更新，则更新已有记录而非新增
- **验收标准**：对同一配置多次生成分享码，列表中仅保留1条记录
