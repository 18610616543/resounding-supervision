## 经验总结

1. **跨模块协作需关注合并顺序**：B 的 PR 基于旧版 main 合并后覆盖了 E 的代码，需通过 hotfix 恢复。教训：合并前应检查 diff 是否删除了其他模块的改动。
2. **分享码健壮性**：原始代码只有一句"未找到该分享码"，无法定位问题。5 种分类错误提示可帮助用户快速排查。
3. **去重设计**：按 name+gender 而非单纯 code 值去重，可以正确处理同一智能体的配置变更场景。
