## 任务清单

### 任务 D-1：代码通读与数据调研

- **目标**：深入理解对话数据库和情感分析算法，找到扩展方向
- **预计时间**：30分钟
- **具体步骤**：
  1. 阅读 `echo-care.html` 中 `dialogueDB`（L440-582）：理解话题数据结构（topic/keywords/replies/emotion）
  2. 阅读 `analyzeEmotion()`（L768-815）：理解关键词层级匹配 + 权重计算逻辑
  3. 阅读 `findReply()`（L818-840）：理解按强度分档取回复的逻辑
  4. 阅读 `generateContextReply()`（L843-859）：理解上下文情感趋势检测
  5. 统计现有话题覆盖范围，列出缺失的常见场景
- **验收标准**：写出对情感分析算法的理解，列出至少2个待新增的话题

---

### 任务 D-2：新增2个话题类别

- **目标**：扩展对话数据库，覆盖更多日常场景
- **预计时间**：40分钟
- **具体步骤**：
  1. 新增话题1（如 **学习/考试**）：
     - 关键词3组（轻度/中度/重度）：如 `['学习','复习','看书','考试','作业']` / `['挂科','考砸','复习不完']` / `['不想学了','学不进去']`
     - 回复分档3组（light/medium/strong），每组至少2条
     - emotion 映射：轻度→default，中度→sad
  2. 新增话题2（如 **宠物**）：
     - 关键词：`['猫','狗','宠物','喵','汪','撸猫','遛狗','可爱']`
     - 回复至少6条（按强度分档）
     - emotion 映射：light→happy
  3. 格式严格对齐现有 `dialogueDB` 条目结构
- **验收标准**：新增话题的 JSON 格式与现有条目一致，关键词不过度宽泛导致误匹配

---

### 任务 D-3：修复至少1个情感匹配 Bug

- **目标**：修复关键词交叉匹配导致的情绪误判
- **预计时间**：30分钟
- **候选 Bug**：
  1. **"喜欢"同时匹配 love 和 greeting**：检查 `analyzeEmotion()` 中关键词打分逻辑是否存在权重不合理交叉
  2. **长文本中多情绪混淆**：用户一句话同时含正面和负面词汇时，检查最强权重情绪是否合理
  3. **英文混合中文文本的识别遗漏**：检查 `lowerText` 转换是否丢失了关键英文情感词
- **具体步骤**：
  1. 选中一个 Bug，构造复现用例（输入特定文本）
  2. 在控制台打印 `analyzeEmotion()` 的返回值，确认误判
  3. 定位并修复
  4. 用原用例验证修复有效
- **验收标准**：修复后原误判用例返回正确的 emotion 和合理的 intensity

---

### 任务 D-4：聊天页配置可定制化

- **目标**：将硬编码的角色名称和标题改为可配置
- **预计时间**：20分钟
- **具体步骤**：
  1. 在 `echo-care.html` 顶部（script 区域开头）增加一个配置对象：
     ```javascript
     const CHAT_CONFIG = {
       title: 'Echo Care',
       characterName: '小艾',
       greeting: '嗨！我是小艾 👋\n我能感知你的情绪，给你贴心的回应～'
     };
     ```
  2. 将页面中硬编码的"小艾"替换为 `CHAT_CONFIG.characterName`
  3. 将初始欢迎语替换为 `CHAT_CONFIG.greeting`
  4. 更新页面 `<title>` 为 `CHAT_CONFIG.title`
- **验收标准**：修改配置对象即可改变角色名和欢迎语，无需搜索替换多处硬编码
